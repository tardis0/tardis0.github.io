From JUnit 4 to JUnit 5

1. Add JUnit 5 API

Adding JUnit 5 support to your existing project that use JUnit 4 is very straightforward because JUnit 5 and JUnit 4 can co-exists in the same project.

The first thing to do is to add JUnit 5 API to your pom.xml

Once added, you can start writing JUnit 5 tests. JUnit 5 classes can be found in package org.junit.jupiter

Running The Tests

Running The Tests on IDE

IntelliJ has great support for JUnit 5, adding JUnit 5 is enough for IntelliJ to discover and run the unit tests.

Running The Tests using Maven

If you are running mvn test, you might notice that the JUnit 5 tests are not run (either by purposely failing a JUnit 5 unit tests or observing the test count in the output)

Run mvn test again. This time tests run sucessfully, but you will notice that the test count still doesnâ€™t look right. The side effect of adding JUnit 5 test provider and test engine will exclude JUnit 4 tests. We can fix this by adding test engine that support JUnit 4 tests (junit-vintage-engine).

Summary

JUnit 4 is a solid test library and will be around for a long time. JUnit 5 is a natural progression of JUnit 4 and by making the migration path painless, very soon JUnit 5 will gain traction.
